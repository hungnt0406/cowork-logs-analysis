Cowork Workflow Miner — Kiến trúc hợp nhất

Hợp nhất 2 dự án: lõi = cowork-logs-analysis (segmentation → LLM judge → calibration → mine), vỏ = cowork-behavior-harness (privacy gate · business sidecar · convergence · skill-draft). TechStack: TypeScript / Bun / SQLite.

Lõi gốc (logs-analysis) Vỏ hợp nhất (từ harness) PRIVACY GATE (fail-closed)

1. Luồng hệ thống (end-to-end)

Liệt kê session Claude Code thật (loại fork/agent-mode/observer).

opt-out filter VỎ · bảo vệ NLĐ
Loại session người lao động đã opt-out (.cwbh-exclude / path personal·private) TRƯỚC khi đọc nội dung.

View gọn ≤12k ký tự cho judge. Credential → DROP, PII → MASK ngay trước khi text rời máy tới LLM.

Rubric chống Claude-grade-Claude, neo vào user-observable behavior. Cache-key đa thành phần, retry, circuit-breaker, cost-cap.

calibrate (cổng tin cậy) CORE
Stratified human spot-check + self-consistency + auto cross-check. (Reconstruct cũng đi qua PRIVACY GATE.)
src/calibrate.ts
mine (cluster + good/bad) CORE
Cluster task_type, contrast pattern tốt/xấu, ranking component (no composite score), Laplace smoothing, "insufficient evidence".
src/mine.ts
business sidecar (join LATE) VỎ
Thêm business note SAU khi đã rank. Judge & mine business-blind để giữ tính độc lập đánh giá.
src/sidecar.ts → report.ts (--business)
report + candidates.json CORE
Báo cáo exemplar-driven (Go/Kill gate). Cũng xuất candidates_<machine>.json cho convergence.
src/report.ts → out/report.md · out/candidates.json
converge VỎ · meta gate
Gom candidates nhiều máy → workflow xuất hiện ≥2 máy = cross-validated.
src/converge.ts → out/convergence.md
skill-draft + E2E test VỎ · gated
Opt-in (--yes) → SKILL.md + run.ts + bun test + REVIEW.md vào review queue. Self-eval. KHÔNG publish.
src/skilldraft.ts → out/skill_drafts/<slug>/
🔒 Privacy là cổng fail-closed: mọi điểm text rời máy tới LLM (render→judge, calibrate-reconstruct→judge) đều đi qua privacy.sanitizeRendered(). Credential bị DROP, PII bị MASK. Audit ghi tại out/state/audit.json.

3. Ràng buộc — enforce ở đâu

Giữ từ lõi logs-analysis

  • AB trên log thật — discover đọc transcript thật, judge bằng LLM thật.
  • Phân biệt tốt/xấu đáng tin — rubric chống thiên vị + calibration trust gate.
  • Chỉ gợi ý — dừng ở Go/Kill gate (candidates.json), không tự tạo.
  • Trung thực N nhỏ — no composite score, Laplace smoothing, "insufficient evidence".

Thêm từ vỏ harness

  • Bảo vệ NLĐ & dữ liệu (API/Pass) — privacy gate fail-closed + opt-out + POLICY + audit.
  • Business = sidecar, join late — judge/mine business-blind; note ghép sau.
  • Hội tụ nhiều máy — converge → cross-validated (open gate cho meta).
  • Full quy trình tạo skill + E2E test — skilldraft gated + self-eval, review queue.
